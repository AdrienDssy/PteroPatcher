# 🦕 Ptero-Addons auto update

An add-on management system for **Pterodactyl Panel**, allowing you to automatically reapply all changes after updating the panel.

---

## 📁 Directory structure

```
/opt/ptero-addons/
├── ainx/             ← Add-ons in .ainx format
├── files/            ← All your files (modified AND custom), mirrored from the panel
├── patches/          ← Generated automatically by generate.sh (modified files)
├── custom/           ← Automatically generated by generate.sh (custom/new files)
├── generate.sh       ← To be run after applying your changes
├── apply.sh          ← To be run after every Pterodactyl update
└── README.md
```

---

## ⚙️ How it works

`generate.sh` scans every file in `files/` and sorts them automatically:

| Case | Result |
|---|---|
| File **already in the panel** and modified | → Patch generated in `patches/` |
| File **not present in the panel** (custom/new) | → Copied to `custom/` |
| File identical to the original | → Ignored `[SKIP]` |

`apply.sh` reapplies in the following order:
1. The `.ainx` add-ons
2. The custom files (direct copy)
3. Patches (modifications to existing files)
4. Rebuild assets + clear cache

---

## 🚀 Initial installation

### 1. Create the directory structure
```bash
mkdir -p /opt/ptero-addons/{ainx,files,patches,custom}
```

### 2. Copy the scripts
```bash
cp generate.sh apply.sh /opt/ptero-addons/
chmod +x /opt/ptero-addons/*.sh
```

### 3. Place the ainx add-ons
```bash
cp *.ainx /opt/ptero-addons/ainx/
```

---

## ✏️ Adding an add-on

### Ainx add-on
Simply place the `.ainx` file in the `ainx/` folder:
```bash
cp my-addon.ainx /opt/ptero-addons/ainx/
```

### Add-on with modifications to existing files
```bash
# 1. Copy the original file from the panel
cp /var/www/pterodactyl/app/Http/Controllers/MyController.php \
   /opt/ptero-addons/files/app/Http/Controllers/MyController.php

# 2. Apply the author’s changes to the copied file
nano /opt/ptero-addons/files/app/Http/Controllers/MyController.php

# 3. Regenerate the patches
bash /opt/ptero-addons/generate.sh
```

### Add-on with custom files (new files)
```bash
# Place the file respecting the panel’s directory structure
cp MyNewController.php \
   /opt/ptero-addons/files/app/Http/Controllers/MyNewController.php

# Regenerate
bash /opt/ptero-addons/generate.sh
```

## 🔄 Updating Pterodactyl

```bash
# 1. Updating the dashboard
cd /var/www/pterodactyl
php artisan down
git pull
composer install --no-dev --optimize-autoloader
php artisan migrate --force

# 2. Reapplying all add-ons
bash /opt/ptero-addons/apply.sh
```

---

## ⚠️ Conflict management

A conflict occurs when Pterodactyl has modified the same file as you during an update.  
`apply.sh` tells you exactly which patch has failed.

**To resolve a conflict:**
```bash
# 1. Retrieve the new official version of the file
cp /var/www/pterodactyl/app/Http/Controllers/MyController.php \
   /opt/ptero-addons/files/app/Http/Controllers/MyController.php

# 2. Manually reapply the addon’s changes to this file
nano /opt/ptero-addons/files/app/Http/Controllers/MyController.php

# 3. Regenerate the patch
bash /opt/ptero-addons/generate.sh

# 4. Reapply the add-ons
bash /opt/ptero-addons/apply.sh
```

## 📋 Quick reference

| Action | Command |
|---|---|
| Generate patches after changes | `bash /opt/ptero-addons/generate.sh` |
| Reapply all add-ons | `bash /opt/ptero-addons/apply.sh` |
| Add a file to the tracker | Copy to `files/` then run `generate.sh` |
| Resolve a conflict | Re-edit in `files/` then run `generate.sh` |
